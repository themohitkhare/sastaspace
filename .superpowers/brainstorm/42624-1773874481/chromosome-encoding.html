Chromosome Encoding for Strip Packing

How should the GA represent a packing arrangement internally?

A

Placement Order + Rotation

Chromosome = [piece_index, rotation_angle] pairs defining the order pieces are placed. A deterministic decoder (bottom-left-fill) converts the order into actual (x, y) positions.

// Chromosome (7 pieces):
[piece_3@90°, piece_1@0°, piece_5@270°, piece_0@180°, ...]
// Decoder places each piece bottom-left-fill in this order
// → deterministic (x,y) positions

Pros

  • Always produces valid packings (no overlaps)
  • Standard approach in literature
  • Crossover = reorder pieces

Cons

  • Decoder quality limits solution quality
  • Bottom-left-fill for irregular polygons is complex
B

Direct Placement (x, y, θ per piece)

Chromosome stores the actual position and rotation of every piece. Fitness penalizes overlaps. More expressive but can produce invalid (overlapping) solutions.

// Chromosome (7 pieces):
[(x=12, y=5, θ=90°), (x=0, y=0, θ=0°), (x=30, y=12, θ=45°), ...]
// May overlap! Fitness = area_used - overlap_penalty

Pros

  • Most expressive — can find solutions decoder can't
  • Visually dramatic (pieces sliding through each other, then separating)
  • Simpler chromosome structure

Cons

  • Many infeasible solutions early on
  • Overlap detection for polygons needed
  • Slower convergence
C

Hybrid: Order + NFP (No-Fit Polygon)

Chromosome = placement order + rotation. Decoder uses No-Fit Polygons (NFP) to find the lowest valid position for each piece. Industry standard for irregular nesting.

// Chromosome (7 pieces):
[piece_3@90°, piece_1@0°, piece_5@270°, ...]
// NFP decoder: for each piece, compute NFP against all placed
// pieces, find lowest-leftmost valid position

Pros

  • Best packing quality — tight fits around irregular shapes
  • Always valid (no overlaps)
  • Used by commercial nesting software

Cons

  • NFP computation is complex to implement
  • Slower per-evaluation
  • Significant math (Minkowski sums)

Recommendation: A (Order + Rotation with bottom-left-fill) — Best balance of quality, implementation speed, and visual appeal. Always produces valid packings. We can upgrade to C (NFP) later for tighter irregular fits.
