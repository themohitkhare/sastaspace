<objective>
Replace local Playwright Chromium launch with a remote Browserless service accessed via Chrome DevTools Protocol (CDP) WebSocket.

Purpose: Decouple browser execution from the backend/worker pods. This eliminates Chromium system deps from the backend image (smaller, faster builds), enables browser pooling, and makes crawling more reliable in k8s.

Output: Browserless k8s manifest, updated crawler.py using CDP, slimmed backend Dockerfile.
</objective>

<task type="auto">
  <name>Task 2: Refactor crawler.py to use remote CDP connection</name>
  <files>sastaspace/crawler.py</files>
  <action>
1. Replace the `_browser_context()` async context manager to connect via CDP instead of launching local Chromium:
   ```python
   @contextlib.asynccontextmanager
   async def _browser_context(browserless_url: str | None = None):
       """Connect to remote Browserless via CDP. Falls back to local launch for dev."""
       async with async_playwright() as pw:
           if browserless_url:
               browser = await pw.chromium.connect_over_cdp(browserless_url)
           else:
               browser = await pw.chromium.launch(headless=True)
           try:
               ctx = await browser.new_context(
                   user_agent=_USER_AGENT,
                   viewport={"width": 1280, "height": 800},
               )
               page = await ctx.new_page()
               yield browser, ctx, page
           finally:
               await browser.close()
   ```

2. Update `crawl()` to accept an optional `browserless_url` parameter and pass it through:
   ```python
   async def crawl(url: str, browserless_url: str | None = None) -> CrawlResult:
   ```
   - Remove the `_ensure_chromium()` call when `browserless_url` is provided (still call it for local fallback)
   - Pass `browserless_url` to `_browser_context()`

3. Update `enhanced_crawl()` similarly:
   - It already receives `settings` — use `settings.browserless_url` if present
   - Pass to `_browser_context(browserless_url=settings.browserless_url)`
   - Skip `_ensure_chromium()` when browserless_url is set

4. Keep the `_ensure_chromium()` function — it is still needed for local dev without Browserless.

5. Do NOT change any extraction logic (`_crawl_page`, `_crawl_internal_page`, `_extract_*` functions). All page interaction code stays identical — only the browser connection method changes.

6. Keep the `from playwright.async_api import async_playwright` import — it is still needed for both CDP and local modes.
