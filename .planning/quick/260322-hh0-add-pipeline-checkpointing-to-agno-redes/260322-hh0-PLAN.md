<objective>
Add pipeline checkpointing so recovered jobs resume from the last completed step instead of restarting the entire crawl-redesign-deploy pipeline from scratch.

Purpose: Each agent step takes 30-60s (7 steps total). When a worker dies mid-pipeline, recovery currently restarts everything. Checkpointing saves ~3-5 minutes per recovered job.

Output: Modified pipeline that saves intermediate results to MongoDB after each major step and resumes from checkpoint on recovery.
</objective>

**jobs.py â€” redesign_handler changes:**

1. After the crawl step succeeds, serialize `crawl_result` minimally (just enough for the pipeline to not need it â€” actually, `crawl_result` IS needed by the pipeline as input, so we need to persist it too). Strategy: after crawl, pickle or JSON-serialize the CrawlResult dataclass fields into a dict and save to the job checkpoint. Use a helper `_serialize_crawl_result(cr: CrawlResult) -> dict` that captures: `title`, `meta_description`, `colors`, `fonts`, `sections`, `html`, `screenshot_base64`, `url`, `links`, `nav_links`, `images`, `error`. And `_deserialize_crawl_result(data: dict) -> CrawlResult` to reconstruct.

2. Save crawl checkpoint after crawl:
```python
crawl_data = _serialize_crawl_result(crawl_result)
await update_job(job_id, checkpoint={"completed_step": "crawl", "crawl_result": crawl_data, "pipeline_data": {}})
```

3. Pass a `checkpoint_callback` to `run_redesign_pipeline` (via `run_redesign` â€” needs a small signature change in `redesigner.py`'s `run_redesign` and `agno_redesign` to accept and forward `checkpoint_callback`). The callback is a closure that calls `update_job` with the new checkpoint data. Since `run_redesign_pipeline` runs in a thread (`asyncio.to_thread`), the callback must schedule the async `update_job` on the event loop, same pattern as `_on_agent_progress`:

```python
def _on_checkpoint(step_name: str, checkpoint_data: dict) -> None:
    full_checkpoint = {
        "completed_step": step_name,
        "crawl_result": crawl_data,
        "pipeline_data": checkpoint_data,
    }
    try:
        asyncio.run_coroutine_threadsafe(
            update_job(job_id, checkpoint=full_checkpoint), loop
        )
    except Exception:
        pass
```

4. In `_recover_pending` and `process_messages` handler call: before calling `handler(job_id, url, tier, job_service)`, load the job via `get_job(job_id)` and check for a `checkpoint` field. If present, pass it to the handler. Modify `redesign_handler` signature to accept an optional `checkpoint: dict | None = None` parameter.

5. In `redesign_handler`, when `checkpoint` is provided:
   - If `checkpoint["completed_step"]` is `"crawl"` or later: skip the crawl step, reconstruct `crawl_result` from `checkpoint["crawl_result"]` using `_deserialize_crawl_result`.
   - Pass `checkpoint.get("pipeline_data")` to `run_redesign` which forwards it to `run_redesign_pipeline`.
   - If `checkpoint["completed_step"]` is only `"crawl"` (no pipeline_data yet): skip crawl, start redesign from beginning.

6. After deploy succeeds, clear the checkpoint: `await update_job(job_id, checkpoint=None)` â€” job is done, no need to keep checkpoint data.
