**Step 1: Create `web/src/lib/sse-client.ts`** (per PROG-02, D-09):

Export a type and an async generator function:

```typescript
export type SSEEvent = {
  event: string;
  data: Record<string, unknown>;
};

export async function* streamRedesign(
  url: string,
  apiBase: string = "http://localhost:8080"
): AsyncGenerator<SSEEvent>
```

Implementation:
- POST to `${apiBase}/redesign` with `Content-Type: application/json`, body `JSON.stringify({ url })`
- If `!response.ok`: parse error body with `response.json().catch(() => ({}))`, throw `new Error(errorBody.error || errorBody.detail || \`HTTP ${response.status}\`)`
- Pipe response body through `new TextDecoderStream()`, get reader
- Buffer incoming text, split on `"\n\n"` (double newline = event boundary)
- For each complete event block, parse `event:` and `data:` lines
- `yield { event: eventName, data: JSON.parse(dataStr) }` for each valid event
- Keep remainder in buffer for next chunk (handles partial chunk delivery)
- Wrap JSON.parse in try/catch to skip malformed events silently

**Step 2: Create `web/src/hooks/use-redesign.ts`** (per PROG-01, PROG-03, PROG-06):

```typescript
export type RedesignState =
  | { status: "idle" }
  | { status: "connecting" }
  | { status: "progress"; currentStep: string; domain: string; steps: StepState[] }
  | { status: "done"; subdomain: string; originalUrl: string; domain: string }
  | { status: "error"; message: string; url: string };

type StepState = {
  name: string;
  label: string;
  value: number; // 0-100
  status: "pending" | "active" | "done";
};
```

Export `useRedesign()` hook returning `{ state: RedesignState, start: (url: string) => void, retry: () => void, reset: () => void }`.

Step definitions (per D-08, D-09 -- NEVER show technical SSE event names to user):
```typescript
const STEPS = [
  { name: "crawling", label: (d: string) => `Analyzing ${d}` },
  { name: "redesigning", label: (_d: string) => "Redesigning your site with AI" },
  { name: "deploying", label: (d: string) => `Preparing your new ${d}` },
] as const;
```

`start(url)` function:
1. Extract domain via `extractDomain(url)` from `@/lib/url-utils`
2. Set state to `{ status: "connecting" }`
3. Initialize steps array with all pending, value 0
4. Set state to `{ status: "progress", currentStep: "crawling", domain, steps }`
5. Iterate over `streamRedesign(url)` async generator in a try block
6. On each event:
   - If event is `"crawling"`, `"redesigning"`, or `"deploying"`: update the matching step to `active` with an intermediate value (crawling: 70, redesigning: 50, deploying: 60), mark all preceding steps as `done` with value 100. Set `currentStep` to this event name.
   - If event is `"done"`: set all steps to done/100, pause 800ms (`await new Promise(r => setTimeout(r, 800))`), then set state to `{ status: "done", subdomain: data.subdomain, originalUrl: url, domain }`
   - If event is `"error"`: set state to `{ status: "error", message: data.error as string, url }`
7. In catch block: set state to `{ status: "error", message: "We couldn't redesign that site right now. This can happen with very large or complex websites.", url }`
8. Use `useRef` to store an `AbortController` for cleanup; abort on unmount via `useEffect` cleanup

`retry()` function: calls `start(state.url)` if state is error
`reset()` function: sets state to `{ status: "idle" }`

**Step 1: Create `web/src/components/app-flow.tsx`** (per D-17, D-18, D-19, PROG-01):

`"use client"` directive.

This component is the state machine orchestrating the three views. It manages the flow: landing -> progress -> result navigation.

```typescript
"use client";
import { useCallback, useEffect } from "react";
import { useRouter } from "next/navigation";
import { AnimatePresence, motion } from "motion/react";
import { HeroSection } from "@/components/landing/hero-section";
import { HowItWorks } from "@/components/landing/how-it-works";
import { ProgressView } from "@/components/progress/progress-view";
import { useRedesign } from "@/hooks/use-redesign";
```

Export `AppFlow` component:
- Call `useRedesign()` to get `{ state, start, retry, reset }`
- Call `useRouter()` for navigation
- `handleSubmit` callback: calls `start(url)`
- `useEffect` watching for `state.status === "done"`: when done, call `router.push(\`/${state.subdomain}\`)` to navigate to the result page. This pushes a history entry so back button goes to `/` (landing), per D-18. The progress view at `/` never pushed its own history entry, per D-19.
- `handleRetry` callback: calls `retry()`
- `handleReset` callback: calls `reset()`

Render logic with `AnimatePresence mode="wait"`:
- When `state.status === "idle"`: render landing view (key="landing"):
  ```tsx
  <motion.div key="landing" initial={{ opacity: 0 }} animate={{ opacity: 1 }} exit={{ opacity: 0, y: -20 }} transition={{ duration: 0.3, ease: "easeInOut" }}>
    <HeroSection onSubmit={handleSubmit} />
    <section className="py-12 px-4">
      <HowItWorks />
    </section>
  </motion.div>
  ```
- When `state.status` is `"connecting"`, `"progress"`, or `"error"`: render progress/error view (key="progress"):
  ```tsx
  <motion.div key="progress" initial={{ opacity: 0, y: 20 }} animate={{ opacity: 1, y: 0 }} exit={{ opacity: 0, y: -20 }} transition={{ duration: 0.3, ease: "easeInOut" }}>
    <ProgressView state={state} onRetry={handleRetry} onReset={handleReset} />
  </motion.div>
  ```
- When `state.status === "done"`: render nothing (user is being navigated away via router.push)
