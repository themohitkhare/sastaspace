# Phase 3 — UI Design Contract

> Visual and interaction contract for the core user-facing flow: landing page, progress experience, and result page. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase scope:** Build the complete flow from URL input through SSE progress to result reveal. Three views (Landing, Progress, Result) orchestrated by a state machine. Contact form is Phase 4.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn (initialized, `components.json` exists) |
| Preset | base-nova, base color neutral, oklch CSS variables |
| Component library | @base-ui/react (shadcn v4 default) + radix-ui (for Progress primitive) |
| Icon library | lucide-react (`Globe`, `ArrowRight`, `Check`, `AlertCircle`, `ExternalLink`, `RotateCcw`) |
| Font | Inter (via `next/font/google`, already wired in root layout) |
| Animation | motion v12.38.0 (import from `motion/react`, already installed) |

**Existing components:** `button.tsx` only. Phase 3 adds `input`, `label`, `progress`, `spotlight`, and all view components.

Declared values (must be multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline padding |
| sm | 8px | Compact element spacing, icon-to-text gap |

| 2xl | 48px | Major section breaks (hero to "how it works") |

## Typography

| Role | Size | Weight | Line Height | Phase 3 Usage |
|------|------|--------|-------------|---------------|
| Body | 16px | 400 (regular) | 1.5 | Subheadline, "how it works" descriptions, status line, result copy |
| Label | 14px | 400 (regular) | 1.4 | URL input placeholder, step labels, "View original site" link |
| Heading | 24px | 600 (semibold) | 1.2 | "How it works" section title, result page header, step section headers |
| Display | 40px | 600 (semibold) | 1.1 | Hero headline (mobile: 28px), "Your new [domain] is ready" |

**Font family:** Inter, loaded via `next/font/google` with `subsets: ["latin"]`.

**Weight update from Phase 2:** Phase 2 used 400/700. Phase 3 shifts to 400/600 (regular + semibold). Semibold (600) provides sufficient hierarchy contrast while feeling more refined and agency-like per D-04 tone requirement. This is the weight used going forward.

**Responsive Display:** The Display role scales down to 28px on viewports below 640px (`text-[28px] sm:text-[40px]`).

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant (60%) | `oklch(1 0 0)` / `--background` | Page background, hero surface, progress view background |
| Secondary (30%) | `oklch(0.97 0 0)` / `--muted` | "How it works" section background, step indicator track, iframe overlay background |
| Accent (10%) | `oklch(0.205 0 0)` / `--primary` | "Redesign My Site" submit button, "Take me to the future" CTA button, active progress bar fill, focused URL input ring |
| Destructive | `oklch(0.577 0.245 27.325)` / `--destructive` | Error state text, error icon fill |

Accent reserved for:
- "Redesign My Site" submit button (hero CTA)
- "Take me to the future" button on result page
- Active/completed progress bar indicator fill (`bg-primary`)
- URL input focus ring (`ring-ring`)
- "Try again" retry button on error state

**Muted foreground** `oklch(0.556 0 0)` / `--muted-foreground`: Used for subheadline text, "how it works" step descriptions, "View original site" link, URL input placeholder text.

**Foreground** `oklch(0.145 0 0)` / `--foreground`: Used for hero headline, status line text, result page header, step labels.

## Interaction Contract

### Page Transitions (D-16 -- Claude's discretion)

| Transition | Animation | Duration | Easing |
|------------|-----------|----------|--------|
| Landing to Progress | Fade out (opacity 1->0, y 0->-20), then fade in (opacity 0->1, y 20->0) | 300ms each | ease-in-out |
| Progress to Result | `router.push("/{subdomain}")` -- full navigation, result page animates in | 400ms | ease-out |
| Error to Landing | Fade transition via AnimatePresence | 300ms | ease-in-out |
| Page load (landing) | Spotlight fades in (1500ms), content fades in (500ms, 200ms delay) | staggered | ease-out |
| Page load (result, shareable) | Content fades in + scales up from 0.95 | 400ms | ease-out |

All transitions use Motion `AnimatePresence` with `mode="wait"` to ensure exit completes before enter begins.

### URL Input Behavior

| Action | Behavior |
|--------|----------|
| Type URL | Attempt favicon fetch after 500ms debounce; show Globe icon until favicon loads |
| Submit empty | Show validation error "Please enter a valid website address" |
| Submit invalid URL | Show validation error "Please enter a valid website address" |
| Submit valid URL | Transition to progress view, begin SSE connection |
| Auto-prepend protocol | If input lacks `http://` or `https://`, prepend `https://` before validation |

### SSE Connection Behavior

| Event | UI Response |
|-------|-------------|
| Connection established | Progress view shown, step 1 bar begins animating |
| `crawling` event | Status line: "Analyzing {domain}", step 1 bar fills |
| `redesigning` event | Status line: "Redesigning your site with AI", step 1 bar complete, step 2 bar fills |
| `deploying` event | Status line: "Preparing your new {domain}", step 2 bar complete, step 3 bar fills |
| `done` event | Status line: "Your redesign is ready!", all bars complete, 800ms pause, then navigate to `/{subdomain}/` |
| `error` event | Transition to error state with message and retry button |
| Network failure | Transition to error state with generic message and retry button |
| Retry clicked | Reset to landing state with URL pre-filled |

### Back Button Behavior (D-18)

| Current Page | Back Navigates To | Implementation |
|-------------|-------------------|----------------|
| Result (`/{subdomain}/`) | Landing (`/`) | Normal browser back -- progress never pushed history |
| Progress (at `/`) | Previous page (before SastaSpace) | Progress is a state overlay, no history entry |

## Spotlight Background Specification

The Spotlight component from aceternity must be adapted for the oklch zinc theme. Default hsla blue gradients are replaced with neutral oklch values.

| Prop | Default (hsla blue) | Adapted (oklch zinc) |
|------|---------------------|----------------------|
| `gradientFirst` | `hsla(210, 100%, 85%, .08)...` | `radial-gradient(68.54% 68.72% at 55.02% 31.46%, oklch(0.8 0 0 / 0.08) 0, oklch(0.5 0 0 / 0.02) 50%, oklch(0.4 0 0 / 0) 80%)` |
| `gradientSecond` | `hsla(210, 100%, 85%, .06)...` | `radial-gradient(50% 50% at 50% 50%, oklch(0.8 0 0 / 0.06) 0, oklch(0.5 0 0 / 0.02) 80%, transparent 100%)` |
| `gradientThird` | `hsla(210, 100%, 85%, .04)...` | `radial-gradient(50% 50% at 50% 50%, oklch(0.8 0 0 / 0.04) 0, oklch(0.4 0 0 / 0.02) 80%, transparent 100%)` |
| `duration` | 7 (seconds) | 7 (keep default -- subtle, not distracting) |
| `translateY` | -350 | -350 (keep default) |

The adapted Spotlight renders as subtle, slowly-moving neutral light beams against the white background -- dynamic enough to feel alive (D-01) but not overwhelming or colorful.
