# Phase 4 — UI Design Contract

> Visual and interaction contract for the contact form and mobile polish pass. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase scope:** Add a lead-capture contact form below the result page's redesign preview, wire it to Resend via a Next.js Route Handler, protect it with invisible Cloudflare Turnstile + honeypot, and do a targeted mobile polish pass across all three pages (landing, progress, result). No new animations, typography changes, or color changes beyond what Phase 3 established.

## Interaction Contract

### Form Submission Flow

| Action | Behavior |
|--------|----------|
| Fill fields | Standard controlled inputs via useState |
| Submit with empty fields | Show per-field error text below each empty field in `text-sm text-destructive mt-2` |
| Submit with invalid email | Show "Please enter a valid email address" below email field |
| Submit with valid data | Button changes to "Sending..." with Loader2 spinner (animated `animate-spin`), all fields disabled |
| Successful submission | AnimatePresence swap: form fades out, thank-you message fades in. No toast, no redirect (D-08) |
| Server error (500) | Show "Something went wrong. Please try again." below submit button as `text-sm text-destructive mt-2`. Button returns to "Send Message". Fields remain filled. |
| Turnstile failure | Show "Verification failed. Please refresh and try again." below submit button |
| Honeypot triggered | Server returns `{ ok: true }` silently -- client shows thank-you state (bot doesn't know it was caught) |

### Form State Machine

```
idle -> submitting -> success
                   -> error -> idle (user fixes and resubmits)
```

| State | Button Label | Fields Enabled | Visible |
|-------|-------------|----------------|---------|
| `idle` | "Send Message" | Yes | Form |
| `submitting` | "Sending..." + Loader2 | No (all disabled) | Form |
| `success` | N/A | N/A | Thank-you message |
| `error` | "Send Message" | Yes | Form + error text |

### Client-Side Validation

| Field | Rule | Error Copy |
|-------|------|------------|
| Name | Non-empty after trim | "Please enter your name" |
| Email | Non-empty + basic email regex (`/.+@.+\..+/`) | "Please enter a valid email address" |
| Message | Non-empty after trim | "Please enter a message" |

Validation runs on submit only (not on blur, not on change). All errors display simultaneously if multiple fields are invalid.

### Loading State Button Specification

| Property | Value |
|----------|-------|
| Label | "Sending..." |
| Icon | `Loader2` from lucide-react, 16px, positioned left of text |
| Animation | `className="animate-spin"` (Tailwind built-in, 1s linear infinite) |
| Disabled | `disabled={true}` during submitting state |

## Mobile Polish Pass (D-19, D-20, D-21)

### Verification Checklist at 375px

| Page | Check | Expected Behavior |
|------|-------|-------------------|
| Landing | URL input + button stacking | Input full width, button full width below (already `sm:flex-row` / `flex-col`) |
| Landing | "How it works" section | Steps stacked vertically with vertical connectors |
| Landing | No horizontal overflow | `overflow-hidden` on hero container prevents `overflow-x` |
| Progress | Step indicators | Full width within `max-w-md`, text wraps gracefully |
| Progress | Status line | Single line or wraps without clipping |
| Result | Iframe aspect ratio | `aspect-[4/3]` on mobile (`sm:aspect-video` on desktop) -- already responsive |
| Result | Contact form fields | All stack vertically, full width within `max-w-xl` |
| Result | Submit button | Full width, 44px height, easy thumb target |
| Result | Divider width | `max-w-3xl` matches content column, does not exceed viewport |

### Common Fix Patterns

| Issue | Fix |
|-------|-----|
| Horizontal overflow on any page | Add `overflow-x-hidden` to the scrolling container or `max-w-full` to the offending element |
| Content touching viewport edge | Ensure `px-4` (16px) minimum horizontal padding on all page-level containers |
| Text too small on mobile | All body text is 16px (no 12px or 13px), all labels are 14px -- verify no overrides |
| Touch target too small | All interactive elements must be at least 44px in height or width |
