  <action>
Audit each page's components for mobile issues at 375px viewport. This is a targeted fix pass — NOT a visual redesign (per D-19, D-21: no new animations, typography changes, or color changes).

**Audit checklist for each component:**

1. **Overflow-x check** (per D-20): Look for elements wider than viewport:
   - Elements with fixed widths (px values > 343px, which is 375 - 32px padding)
   - Elements without `max-w-full` or `overflow-hidden` that could expand
   - Negative margins that push content outside container

2. **Horizontal padding**: Verify all page-level containers have at least `px-4` (16px each side)

3. **Touch targets**: All buttons and interactive elements must be at least 44px tall (`h-11` or `min-h-11`)

4. **Text sizing**: No text smaller than 14px (no `text-xs` on important content). Body at 16px, labels at 14px.

5. **Stacking**: Verify flex containers use `flex-col` on mobile for form fields, buttons

**Specific fixes to apply (ONLY if issues are found during audit):**

- If `hero-section.tsx` has any element that could overflow at 375px: add `overflow-hidden` to the hero container or `max-w-full` to the offending child
- If `progress-view.tsx` step indicators clip text: ensure text has `break-words` or container has adequate width
- If `result-view.tsx` iframe container doesn't have adequate mobile padding: verify `px-4` on the parent
- If `contact-form.tsx` textarea or inputs don't fill width: verify they have `w-full`
- If the `<hr>` divider in result-view extends beyond viewport: ensure it has `max-w-full` as well as `max-w-3xl`

**Do NOT change:**
- Colors, typography scale, animations, or any visual design (D-21)
- Component structure or feature behavior
- Desktop layout

If all components already pass the mobile audit (Phase 3 was built mobile-first), document that no changes were needed in the task output. The goal is verification + targeted fixes, not wholesale changes.
