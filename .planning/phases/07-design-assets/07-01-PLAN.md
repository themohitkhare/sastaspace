  <action>
    **Step 1 — Create Stitch project and generate designs:**
    - Call `mcp__stitch__create_project` with name "SastaSpace Branding" to get a project ID
    - Call `mcp__stitch__generate_screen_from_text` to generate a favicon/icon design:
      - Prompt: "Minimal app icon for 'SastaSpace' — an AI website redesigner tool. Dark background (near-black #1a1a1a), a stylized 'S' lettermark or abstract geometric logo mark in white/light gray with a subtle blue accent (#6366f1). Clean, modern, tech-forward. Square format, works at 16x16 through 512x512. No text besides the mark."
    - Call `mcp__stitch__generate_screen_from_text` to generate an OG image design:
      - Prompt: "Social sharing preview card (1200x630) for 'SastaSpace — AI Website Redesigner'. Dark background (#1a1a1a). Left side: large SastaSpace logo mark. Right side: 'SastaSpace' in bold white sans-serif, tagline 'See your website redesigned by AI in 60 seconds' in gray. Subtle blue accent (#6366f1). Professional, minimal, tech-forward."
    - Use `mcp__stitch__export_component` or appropriate export method to get the generated assets

    **Step 2 — Create/place icon files:**
    If Stitch MCP produces usable image exports, use them directly. If Stitch produces design screens but not exportable icon files, create programmatic SVG-based icons instead:
    - Create `web/src/app/icon.png` — 32x32 PNG (Next.js auto-serves as favicon alternative)
    - Create `web/src/app/apple-icon.png` — 180x180 PNG for iOS
    - Replace `web/src/app/favicon.ico` with the new design
    - Create `web/public/icon-192.png` — 192x192 for Android
    - Create `web/public/icon-512.png` — 512x512 for Android
    - Create `web/public/og-image.png` — 1200x630 OG image

    Fallback approach if MCP export is limited: Use a canvas-based Node.js script (sharp, canvas, or similar) to programmatically generate the icon and OG image from SVG. The design should be a stylized "S" lettermark on dark background with blue accent, consistent across all sizes.

    **Step 3 — Create web manifest:**
    Create `web/src/app/manifest.ts` using the Next.js App Router manifest convention:
    ```typescript
    import type { MetadataRoute } from 'next'

    export default function manifest(): MetadataRoute.Manifest {
      return {
        name: 'SastaSpace - AI Website Redesigner',
        short_name: 'SastaSpace',
        description: 'See your website redesigned by AI in 60 seconds',
        start_url: '/',
        display: 'standalone',
        background_color: '#1a1a1a',
        theme_color: '#1a1a1a',
        icons: [
          { src: '/icon-192.png', sizes: '192x192', type: 'image/png' },
          { src: '/icon-512.png', sizes: '512x512', type: 'image/png' },
        ],
      }
    }
    ```

    **Step 4 — Update layout.tsx metadata:**
    Update `web/src/app/layout.tsx` to include icons and OG image in metadata. Read `node_modules/next/dist/docs/` for any relevant metadata API changes before writing code:
    ```typescript
    export const metadata: Metadata = {
      title: "SastaSpace - AI Website Redesigner",
      description: "See your website redesigned by AI in 60 seconds",
      openGraph: {
        title: "SastaSpace - AI Website Redesigner",
        description: "See your website redesigned by AI in 60 seconds",
        images: [{ url: "/og-image.png", width: 1200, height: 630 }],
      },
      twitter: {
        card: "summary_large_image",
        title: "SastaSpace - AI Website Redesigner",
        description: "See your website redesigned by AI in 60 seconds",
        images: ["/og-image.png"],
      },
    };
    ```
    Note: favicon.ico, icon.png, and apple-icon.png in `app/` are auto-detected by Next.js App Router — no explicit metadata entry needed for those.

    **Important:** Before modifying layout.tsx, check `web/node_modules/next/dist/docs/` for metadata API docs to ensure compatibility with installed Next.js version.
