  <action>
    Create directory `cloudflared/` at repo root and add `config.yml` as a template with placeholder values for tunnel-specific fields. The actual tunnel UUID and hostname require one-time manual setup (`cloudflared tunnel login` + `cloudflared tunnel create sastaspace`).

    Create `cloudflared/config.yml` with this exact content:

    ```yaml
    # Cloudflare Zero Trust tunnel configuration
    # Setup: cloudflared tunnel login && cloudflared tunnel create sastaspace
    # Then replace <TUNNEL_UUID> and <HOSTNAME> below.
    # Run: cloudflared tunnel --config cloudflared/config.yml run

    tunnel: <TUNNEL_UUID>
    credentials-file: ~/.cloudflared/<TUNNEL_UUID>.json

    ingress:
      # API routes -> FastAPI (port 8080)
      - hostname: <HOSTNAME>
        path: ^/api/.*
        service: http://localhost:8080

      # Everything else -> Next.js (port 3000)
      - hostname: <HOSTNAME>
        service: http://localhost:3000

      # Required catch-all rule (Cloudflare requires this)
      - service: http_status:404
    ```

    CRITICAL: FastAPI runs on port 8080 (not 8000). This is set in sastaspace/config.py as the default. The `/api/*` path regex matches all API routes and sends them to FastAPI. All other traffic goes to Next.js on port 3000.

    The `<TUNNEL_UUID>` and `<HOSTNAME>` are placeholders. The user must:
    1. Install cloudflared: `brew install cloudflared`
    2. Login: `cloudflared tunnel login`
    3. Create tunnel: `cloudflared tunnel create sastaspace`
    4. Replace placeholders with actual UUID and hostname
  </action>
