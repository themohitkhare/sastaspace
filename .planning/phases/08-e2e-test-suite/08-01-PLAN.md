# Plan: Playwright E2E Test Suite

## Objective

Create a comprehensive Playwright E2E test suite verifying the full user flow (landing, URL validation, SSE progress, result page, contact form) with Docker test runner support.

## Context

- @web/playwright.config.ts — existing Playwright config (basic, no webServer)
- @web/e2e/sastaspace.spec.ts — existing tests covering landing, result, contact form, validation
- @web/src/components/app-flow.tsx — state machine managing view transitions
- @web/src/hooks/use-redesign.ts — SSE client hook with step states
- @docker-compose.yml — existing Docker Compose with backend, frontend, claude-code-api

## Tasks

### Task 1: Enhance Playwright config, add SSE progress test, and add npm scripts
type="auto"

**Do:**
1. Update `web/playwright.config.ts` to include `webServer` config that starts Next.js dev server automatically
2. Add SSE progress test (TEST-03) to `web/e2e/sastaspace.spec.ts` using Playwright route interception to mock the backend SSE stream — verify progress view appears with step indicators (crawling, redesigning, deploying)
3. Clean up `web/e2e/debug.spec.ts` (remove debug-only test file)
4. Add `test:e2e` npm script to `web/package.json`
5. Verify all existing tests still cover TEST-01 (landing page), TEST-02 (invalid URL), TEST-04 (result page + iframe + contact form), TEST-05 (contact form validation)

**Done when:**
- `web/playwright.config.ts` has `webServer` configured
- SSE progress test exists with route interception
- `npm run test:e2e` script exists in package.json
- All TEST-01 through TEST-05 requirements have test coverage

### Task 2: Add Docker test service
type="auto"

**Do:**
1. Add a `tests` service to `docker-compose.yml` with `profiles: [test]` so it only runs when explicitly requested
2. The test service should build from `web/` context, install Playwright browsers, and run the E2E suite against the frontend service
3. Use `docker compose --profile test run tests` to execute

**Done when:**
- `docker-compose.yml` has a `tests` service with test profile
- Service is configured to run Playwright tests against the frontend container
- TEST-06 requirement is satisfied

## Verification

- All 6 TEST requirements mapped to specific test cases
- `npx playwright test` passes locally (with dev server running)
- Docker test service definition is valid YAML

## Success Criteria

1. Landing page tests verify hero, URL input, how-it-works (TEST-01)
2. Invalid URL shows validation error (TEST-02)
3. Valid URL with mocked SSE triggers progress view with step indicators (TEST-03)
4. Result page renders iframe and contact form (TEST-04)
5. Contact form empty submit shows errors (TEST-05)
6. Docker compose has test service (TEST-06)
